## 0. Operating Rule (read first)

Effective this filing, TimeTribe stops minting one `CO-<N>-<slug>.md` file per change order. **Every past and future change order lives in this one file.**

**Why.** Auditing the 27-file corpus to build this ledger surfaced three failure modes that a single running file structurally prevents:
1. **A numbering collision** ‚Äî two independently-ratified documents both titled themselves CO-26 (¬ß4.1). **A second, same-shape collision (¬ß4.8) surfaced hours later, the day of this same filing** ‚Äî this ledger's own consolidation audit missed a live CO-27 document when resolving the first collision, inadvertently reassigning an already-claimed number. Proof the single-ledger convention below is necessary, not just tidy: even the audit that motivated it wasn't immune until every past CO lived in one place.
2. **Orphaned citations** ‚Äî CO-18, CO-22, CO-23, CO-24 are referenced by number in other ratified documents but have no back-filled standalone source file in project knowledge (¬ß4.2‚Äì4.4).
3. **A stale repo-mount snapshot** ‚Äî the `/mnt/project` copy of CO-20 predates its own ¬ß0.4 audit-resolution amendment; the live, current version disagrees with the mounted one (¬ß4.5).

**Going forward:**
- **New change order** ‚Üí append a new dated entry under ¬ß2, next available number. No new file.
- **Amendment to a past CO** ‚Üí a new `¬ß0`-style amendment note *inside that CO's entry in this ledger*, same in-place-correction pattern CO-20/CO-21 already established. No ruling is silently deleted ‚Äî only corrected or superseded with a note, same as always.
- **After every edit to this file**, log the change via the memory tool so the convention and the latest state survive across sessions without re-reading this file end to end.
- The historical `CO-16` through `CO-29` files remain on disk, unmodified, as the pre-consolidation audit trail. They are not deleted and are no longer the authoritative record going forward ‚Äî this ledger is.
